Define Extensible SSO for Microsoft Authenticator SSO (iOS/ iPadOS)

Add Microsoft URL prefixes to the Extensible Single Sign-On (SSO) configuration so the Microsoft Authenticator app can detect and process authentication requests for supported Microsoft services.

You must:
  1. Assign User Licenses in Microsoft Entra ID/Azure AD
  2. Add a Microsoft Authenticator SSO Payload (iOS/iPadOS)
  3. Connect Microsoft Entra ID for Microsoft Authenticator SSO (iOS/ iPadOS)
  4. Configure Microsoft Access Management for Microsoft Authenticator SSO (iOS/iPadOS)
This is the fifth step of configuring Microsoft Authenticator SSO for your iOS/ iPadOS devices. See Configuring Microsoft Authenticator Single Sign-On (iOS/ iPadOS). Add the Microsoft-defined login URL prefixes to the Extensible SSO tab. These prefixes enable the Authenticator app to recognize authentication requests for SSO processing.
  1. In the Microsoft Authenticator SSO payload, navigate to the Extensible SSO tab.
  2. Select (Add) to enter each of the following Microsoft-provided URL prefixes: 
    https://login.microsoftonline.com
https://login.microsoft.com
https://sts.windows.net
https://login.partner.microsoftonline.cn
https://login.microsoftonline.us
https://login.chinacloudapi.cn (Not required in India)
https://login-us.microsoftonline.com (Not required in India)
  3. Select Save to apply the configuration.
Proceed with configuring Microsoft Authenticator SSO for your iOS/ iPadOS devices by deploying the Microsoft Authenticator SSO payload and application. See Deploy the Microsoft Authenticator SSO Payload and Application (iOS/ iPadOS).
