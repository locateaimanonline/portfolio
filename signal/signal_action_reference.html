Signal Policy Actions

About This Reference

This reference topic provides option details for the third step in the five-step process of configuring a Signal policy, where you define the actions that execute when policy conditions are met. For procedural instructions, see Signal Policy Wizard | Add Policy Actions.

Action Options

Actions determine what the Signal policy does when its conditions are satisfied.

Action Description and Behavior
Relocate Device

Automatically move devices that trigger the policy from their current device group to a different one.
Send Email

Send an email notification indicating that the policy has been triggered. Add recipients to the To, CC, and BCC fields.

Important: Separate recipients with a semicolon, and do not use spaces.

Use the Macro button in the Subject and Message fields to insert dynamic values, such as Device ID or Policy Name.

Warning: Macros may contain sensitive or personal information.
Remember: Custom Attributes and Custom Data must be created before they appear in the macro menu.
Set Custom Attribute

Set the value of a custom attribute on a managed device.

Note: Supports Android Plus, Apple, Linux, Windows Desktop Classic, Windows Mobile/CE, and Windows Modern devices.
Trigger Alert

Send an alert to the SOTI MobiControl notification panel.

Use the Macro button in the Alert Title and Message fields to insert dynamic values.

Remember: Custom Attributes and Custom Data must be created before they appear in the macro menu.

Select a priority:

  • Low
  • Medium (default)
  • High

Alerts appear in the Notifications panel with the selected priority.
Trigger Webhook

Send automated JSON or XML data to a URL when the policy triggers. Select a predefined webhook, specify a subject, and choose source attributes.

Important: Webhooks are defined in Global Settings > Services > Webhook.
Allow Exchange Access

When the Signal policy is triggered, SOTI MobiControl grants the selected devices permission to connect to the Microsoft Exchange Server Enterprise Resource Gateway (ERG).

Note: Supports Android Plus, Apple, Windows Mobile/CE, and Windows Modern devices.
Allow SOTI Hub Access

Enable SOTI Hub access to a device that SOTI MobiControl previously blocked or did not allow access.

Note: Supports Android Plus and Apple devices.
Allow SOTI Surf Access

Enable SOTI Surf access to a device that SOTI MobiControl previously blocked or did not allow access. This action applies only to devices with SOTI Surf installed.

Note: Supports Android Plus and Apple devices.
Block Exchange Access

When the Signal policy triggers, SOTI MobiControl blocks access to the Microsoft Exchange Server Enterprise Resource Gateway for the targeted devices.

Note: Supports Android Plus, Apple, Windows Mobile/CE, and Windows Modern devices.
Block SOTI Hub Access

Temporarily block device users from accessing content in SOTI Hub when the Signal policy is activated.

Note: Supports Android Plus and Apple devices.
Block SOTI Surf Access

Temporarily block device users from accessing SOTI Surf. This action applies only to devices with SOTI Surf installed.

Note: Supports Android Plus and Apple devices.
Check-in

Synchronize the device with the SOTI MobiControl Deployment Server.

Note: Supports Android Plus, Apple, Linux, Windows Desktop Classic, Windows Mobile/CE, and Windows Modern devices.
Clear SOTI Hub Cache

Clear the cache and delete all downloaded content from SOTI Hub on the device, except mandatory files.

Note: Supports Android Plus and Apple devices.
Clear SOTI Surf Cache

Delete all downloaded content from SOTI Surf on the device, except mandatory files. This action applies only to devices with SOTI Surf installed.

Note: Supports Android Plus and Apple devices.
Log Out Shared Device

SOTI MobiControl remotely logs out all active users from the device simultaneously. Logging out signs out all current users.

Note: Supports Android Plus and Apple devices. This action applies only to devices configured as shared devices.
Send Message

Send a message directly to devices targeted by the Signal policy. You can configure the message type (for example, Error or Warning) and limit how long the message displays on the device.

Note: Supports Android Plus, Apple, Windows Desktop Classic, Windows Mobile/CE, and Windows Modern devices.
Send Script

Automatically run scripts on devices when policy conditions are met.

Select an action type:

  • Server Initiated – The device receives a command from SOTI MobiControl before executing the script.
  • Device-Initiated – The device evaluates conditions and executes the script without requiring a connection to SOTI MobiControl. See Device-Side Evaluation Properties.
    Note: Android devices running SOTI MobiControl Agent 2026.0.0 or later support device-initiated scripting when conditions are set to ALL.

Supported script types:

  • Device-initiated: Legacy and JavaScript scripts
  • Server initiated: Legacy, JavaScript, PowerShell 32-bit Compatibility, and PowerShell Native scripts

Select a script from Execute Script, then choose a delivery method:

  • MobiControl Agent – Recommended for online devices actively connected to the server.
  • Platform Notification Service – Use when immediate delivery is required or when the agent is inactive.
  • SMS – Use for devices with limited or no internet access.
Tip: The Execute Script list is populated from the Manage Scripts dialog box.
Note: Supports Android Plus, Apple, Linux, Windows Desktop Classic, Windows Mobile/CE, and Windows Modern devices.
Soft Reset

Perform a soft reset to close all open applications and clear data stored in the device’s RAM.

Note: Supports Android Plus, Apple, Linux, Windows Desktop Classic, Windows Mobile/CE, and Windows Modern devices.
CAUTION: Use platform-independent actions sparingly. To prevent repeated triggers, enable Limit Action Execution when scheduling the policy. See Signal Policy Wizard | Defining Policy Schedules.

Macros in Signal Policy Actions

Macros allow Signal policy actions to insert dynamic device or server values at runtime. When a Signal policy triggers, SOTI MobiControl resolves each macro and replaces it with the current value at execution time.

Macros are supported only in the Send Email and Trigger Alert actions. The macro drop-down displays only the macros that are valid for the selected action. Macros are resolved when the policy triggers, not when the policy is saved. Custom Attributes and Custom Data must already exist before they appear in the macro list.

Important: Macros may resolve to sensitive or personal information. If a macro value is unavailable at execution time, the resolved output may be empty.
Tip: Combine macros with static text to create contextual messages. For example: 
Device: %DEVICEID% (%MODEL% / %PLATFORM%)
                    Group: %DEVICEGROUPPATH%
                    Triggered: %TRIGGEREDCONDITIONS%
                    Time: %TIMESTAMP%
