Signal Policy Wizard | Add Policy Actions

Ensure you have Manage Signal Policies permissions. You must define Signal policy settings before adding actions. See Signal Policy Wizard | Configure Policy Settings.
Note: In policies with two or more Managed Device condition groups, device-specific actions trigger only for members of the device groups defined in those condition groups. The Source Names macro identifies the object or objects that triggered the action. For example, if a condition triggers when a device enters a geofence, the Source Names macro shows the device name and the location.
This is the fourth of five steps in configuring a Signal policy. In this step, you define the actions that execute when policy conditions are met. For an overview of all five steps in the Signal policy workflow, see Signal Policy Wizard Overview.
Note: Only actions compatible with the conditions defined in the conditions sections are made available.
  1. Select Add Action.
  2. Select an action from the Action list.

    For descriptions of each action and supported platforms, see Action Options.

  3. Configure the settings for the selected action.
    Some actions support macros that insert dynamic device or server values at runtime.
    Important: Macros are supported only for the Send Email and Trigger Alert actions. Macros may resolve to sensitive information at runtime. To use Custom Attributes or Custom Data in macros, you must create them first.

    For macro behavior, supported fields, and usage examples, see Macros in Signal Policy Actions.

  4. Select Add to save the action. Repeat to add more actions as required.
    CAUTION: Use platform-independent actions, such as Send Email or Trigger Alert, sparingly. For example, a policy with two condition groups, where either group can often be True, may result in many emails or alerts. Device-specific actions, such as Send Script or Send Message, only trigger for the Managed Devices condition group, but can still occur often. To prevent repeated triggers, enable the Limit Action Execution feature in the scheduling phase of the Signal Policy Wizard.
    This setting ensures actions run only when the condition status changes. For example, if the battery percentage drops below 20%, an alert triggers. However, if the battery remains below 20% at the next check-in, no further action occurs. The alert only triggers again if the battery level rises above 20% and then drops below 20% again, indicating a change in status. See Signal Policy Wizard | Defining Policy Schedules.
After defining actions, select Next to define the Signal policy schedule in the Scheduling section. See Signal Policy Wizard | Defining Policy Schedules.
