Signal Policy Wizard | Build Policy Conditions

Ensure you have Manage Signal Policies permissions. This task continues from creating a new Signal policy. For more information, see Signal Policy Wizard | Create a Policy.
This is the second of five steps in configuring a Signal policy. In this step, you define the conditions that trigger the policy. You create conditions by selecting categories, specifying attributes, and defining values. You can combine more than one condition and group to control when actions run.

For an overview of all five steps in the Signal policy workflow, see Signal Policy Wizard Overview.

In the Conditions tab of the Signal policy wizard:
  1. Select Add Condition.
  2. From the Category drop-down select one for the condition.
    Note: Available subcategories vary based on the category selected. For details about available categories and their subcategories and use cases, see Signal Policy Conditions | Category Details.
  3. Select a subcategory for the selected category.
    Note: Available subcategories vary based on the category selected. For details about available categories and their subcategories and use cases, see Signal Policy Conditions | Category Details.
  4. From the Type drop-down, select a condition-attribute type.
    • Property: Details of devices, servers, or location zones, such as available memory, server name, or device count (within a zone).
    • Event: Details of operational events, such as a SIM card insertion or a deployment server stopping.
  5. Optional: When selecting an a property, choose an Attribute, an Operator, and enter the desired value.
    Tip: Use Evaluate property duration to monitor a compatible device property or event over a continuous period of time instead of reacting to every state change. This helps reduce alert noise and ensures actions trigger only when the condition remains true for a defined duration. To review the list of all device properties and events that can be aggregated, see Signal Policy | Device Properties and Events.
  6. Select Add to add the condition.
    Important: When adding a device property or event, you must also define the target device group(s).
  7. Repeat these steps to add additional conditions as required.
    Tip: Hover over a condition row to edit or delete it.
  8. Optional: If you add multiple conditions, select how many must be met by using the X of these conditions must be satisfied drop-down.
    • ALL: The policy executes actions only when all conditions in the group are met.
    • ANY: The policy executes actions when any condition in the group is met.
  9. Select New Group to add another condition group.
  10. Optional: If you add multiple condition groups, select how groups are evaluated by using the Grouped Conditions to Trigger Actions Must Include drop-down.
    • ALL: The policy executes actions only when all condition groups are met.
    • ANY: The policy executes actions when any condition group is met.
Select Next to continue to the Settings tab. See Signal Policy Wizard | Configure Policy Settings.
